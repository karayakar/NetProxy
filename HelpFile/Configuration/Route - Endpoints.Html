Route - Endpoints



When a connection is made to the NetProxy, its primary function to the make a secondary connection and divert all data from the incoming connection to this new outgoing connection. Endpoints are these outgoing connections. They can be used to load-balance or provide failover.

  • Connection Pattern:
    • Round-Robbin: Round-Robbin will cause each incomming connection to be sent to the next Address in the endpoint pool.
    • Fail-Over: Fail-Over will send all incomming connections into the first endpoint in the pool until connections to that address fail. Upon failure, any incomming connections will be sent to the next address in the pool.
  • Use Sticky Sessions: When enabled, causes any incomming connection from a given IP address "stick" to the endpoint which was originally seleced for it.
  • Remote Endpoints:
    • Enabled: Whether the address is enabled or disables.
    • Address: The IP address or host name of the remote endpoint. Note: if a host name is used, there can be a performance hit in resolving to the IP address.
    • Port: The TCP/IP port of the remote service.
    • Description: A description of the endpoint.
