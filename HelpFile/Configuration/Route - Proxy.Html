Route - Proxy



The proxy settings allow one instance of NetProxy to communicate natively with another instance of NetProxy. This can be useful when ordinary unencrypted data needs to be encrypted while in transit - with the added benefit of data compression. Pre-shared keys are used for all communication over a proxy, but each and every connection made through the proxy will negotiate its own unique pair of send/receive keys using a Diffie-Hellman key exchange.

  • Incomming / Binding:
    • Is Proxy: Whether or not the incomming connection is from an instance of NetProxy.
    • Use Compression: Whether or not data to and from the other instance of NetProxy is to be compressed.
    • Use Encryption: Whether or not data to and from the other instance of NetProxy is to be encrypted.
    • Pre-Shared Key: The pre-shared key for early negoation encryption. Also serves as authentication as both ends of the connection must share the same key.
  • Outgoing / Endpoints:
    • Is Proxy: Whether or not the outgoing connection is to an instance of NetProxy.
    • Use Compression: Whether or not data to and from the other instance of NetProxy is to be compressed.
    • Use Encryption: Whether or not data to and from the other instance of NetProxy is to be encrypted.
    • Pre-Shared Key: The pre-shared key for early negoation encryption. Also serves as authentication as both ends of the connection must share the same key.
