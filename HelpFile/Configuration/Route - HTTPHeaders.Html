Route - HTTPHeaders



The HTTP Headers configuration allows you to manipulate HTTP headers as the requests flow through NetProxy

For instance, if you were routing traffic to an IP address where the webserver is configured to deliver content based on the "HOST:" header, then you would add a rule to add the appropriate host name header to the HTTP header.

  • Type: Whether we are going to apply the rule to incoming requests or outgoing connections.
  • Verb: The HTTP verb to apply the rule to. For example, we might only want to append a header to GET requests.
  • Header: The name of the header field that we want to add, remove or update.
  • Action: The action to take with the header field such as insert, update, delete, etc.
  • Value: If inserting, updating or upserting, this is the value to set the header field to.
  • Description: A place to describe what the rule is intended to do.
